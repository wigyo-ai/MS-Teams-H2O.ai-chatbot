  4. Click Register.
  5. On the overview page, copy and save:
    Application (client) ID → this is your MicrosoftAppId
    Directory (tenant) ID → this is your MicrosoftAppTenantId

B
Generate the Client Secret
  1. In the App Registration, go to Certificates & secrets → + New client secret.
  2. Set a description (e.g. bot-secret) and expiry (12 or 24 months) → Add.
  3. Copy the Value column immediately — it is only shown once.
⚠️
The secret value disappears after you navigate away. Save it to a password manager now — this becomes MicrosoftAppPassword.
C
Create the Azure Bot resource

The Azure Bot resource connects your App Registration to the Bot Framework channels (Teams, etc.).

  1. In the Azure portal → Create a resource → search "Azure Bot" → Create.
  2. Fill in the form:
    • Bot handle: h2o-teams-bot02
    • Subscription: your H2O.ai subscription
    • Resource group: rg-h2o-teams-bot
    • Type of App: Single Tenant
    • Creation type: Use existing app registration
    • App ID: paste <your-app-id> from Phase A
  3. Click Review + create → Create. Deployment takes ~1 minute.
  4. Once deployed, go to the resource → Configuration and confirm the App ID and Tenant ID are correct.

bot/h2o_bot.py

python
import logging
from botbuilder.core import ActivityHandler, TurnContext, MessageFactory
from botbuilder.schema import ChannelAccount
from h2ogpte import H2OGPTE
from config import Config

logger = logging.getLogger(__name__)

# Shared client — one connection to h2oGPTe for all conversations
_client = H2OGPTE(address=Config.H2OGPTE_URL, api_key=Config.H2OGPTE_API_KEY)

# Persist chat sessions per Teams conversation so history is maintained
_sessions: dict[str, str] = {}


def _get_session(conversation_id: str) -> str:
    if conversation_id not in _sessions:
        _sessions[conversation_id] = _client.create_chat_session()
    return _sessions[conversation_id]


class H2OBot(ActivityHandler):

    async def on_message_activity(self, turn_context: TurnContext) -> None:
        user_text = (turn_context.activity.text or "").strip()
        if not user_text:
            return
        conversation_id = turn_context.activity.conversation.id
        try:
            session_id = _get_session(conversation_id)
            with _client.connect(session_id) as session:
                reply = session.query(user_text, timeout=120)
            answer = reply.content
        except Exception as exc:
            logger.exception("h2oGPTe query failed")
            answer = f"Sorry, I encountered an error: {exc}"
        await turn_context.send_activity(MessageFactory.text(answer))

    async def on_members_added_activity(
        self, members_added: list[ChannelAccount], turn_context: TurnContext
    ) -> None:
        for member in members_added:
            if member.id != turn_context.activity.recipient.id:
                await turn_context.send_activity(
                    MessageFactory.text("Hi! I'm your H2O AI assistant. Ask me anything.")
                )

bot/app.py

python
import logging, sys
from aiohttp import web
from botbuilder.core import BotFrameworkAdapter, BotFrameworkAdapterSettings
from botbuilder.schema import Activity
from config import Config
from h2o_bot import H2OBot

logging.basicConfig(stream=sys.stdout, level=logging.INFO,
    format="%(asctime)s %(levelname)s %(name)s: %(message)s")

SETTINGS = BotFrameworkAdapterSettings(
    app_id=Config.APP_ID,
    app_password=Config.APP_PASSWORD,
    channel_auth_tenant=Config.APP_TENANT_ID,  # required for SingleTenant
)
ADAPTER = BotFrameworkAdapter(SETTINGS)
BOT     = H2OBot()

async def on_error(context, error):
    await context.send_activity("An unexpected error occurred.")

ADAPTER.on_turn_error = on_error

async def messages(req: web.Request) -> web.Response:
    if "application/json" not in req.headers.get("Content-Type", ""):
        return web.Response(status=415)
    body     = await req.json()
    activity = Activity().deserialize(body)
    auth     = req.headers.get("Authorization", "")
    await ADAPTER.process_activity(activity, auth, BOT.on_turn)
    return web.Response(status=200)

async def health(_req): return web.Response(text="OK")

app = web.Application()
app.router.add_get("/", health)
app.router.add_post("/api/messages", messages)

if __name__ == "__main__":
    web.run_app(app, host="0.0.0.0", port=Config.PORT)

Part 3 — Azure App Service Deployment
Deploy the bot server to production using Azure CLI

Install Azure CLI first

bash
# macOS
brew install azure-cli

# Verify
az --version
10
Create the Azure App Service
Login → App Service Plan → Web App
›

Login to H2O.ai's Azure tenant

bash
az login --tenant h2o.ai

# Confirm you are on the right subscription
az account show

# Switch if needed
az account set --subscription "<subscription-name-or-id>"

Create App Service Plan

⚠️
Region matters for quota. eastus / westus may have 0 Free VM quota on new subscriptions. westeurope is confirmed working. Always include --is-linux — without it the plan is created as Windows and Python is unsupported.
bash
az appservice plan create --name h2o-bot-plan --resource-group rg-h2o-teams-bot --location westeurope --sku F1 --is-linux

Create the Web App

bash
az webapp create --name h2o-teams-bot02 --resource-group rg-h2o-teams-bot --plan h2o-bot-plan --runtime "PYTHON:3.11"
✅
This generates your permanent production URL: https://h2o-teams-bot02.azurewebsites.net
ℹ️
F1 (Free) has a 60 CPU-min/day limit and sleeps after 20 min idle — fine for testing. Upgrade to --sku B1 (~$13/mo) for production always-on.

Upload Code + Deploy to Azure
Get files → save locally → zip → deploy → set endpoint
›
A
Get the Python files from h2oGPTe Tool Builder

If you generated the bot code using the h2oGPTe Tool Builder agent, download the files first.

  1. In h2oGPTe, open the Tool Builder chat session where you generated the bot code.
  2. Click Details (top right of the chat) → Files tab.
  3. Download each file individually or use Download All:
    app.py  ·  config.py  ·  h2o_bot.py  ·  requirements.txt
ℹ️
If you wrote the files yourself, skip to Phase B. Either way the files must end up in a local bot/ folder before deploying.
B
Save the files into a local bot/ folder
bash — create the folder
mkdir -p ~/teams-h2o-bot/bot

Move your downloaded files into that folder. The final structure should look like:

folder structure
~/teams-h2o-bot/
└── bot/
    ├── app.py
    ├── config.py
    ├── h2o_bot.py
    └── requirements.txt
C
Set environment variables on Azure

These replace the .env file — Azure injects them into the running process.

bash
az webapp config appsettings set --name h2o-teams-bot02 --resource-group rg-h2o-teams-bot --settings MicrosoftAppId="<your-app-id>" MicrosoftAppPassword="<your-client-secret>" MicrosoftAppTenantId="<your-tenant-id>" H2OGPTE_URL="<your-h2ogpte-url>" H2OGPTE_API_KEY="<your-api-key>" SCM_DO_BUILD_DURING_DEPLOYMENT="true"
D
Set the startup command
bash
az webapp config set --name h2o-teams-bot02 --resource-group rg-h2o-teams-bot --startup-file "python app.py"
E
Zip the files and deploy to Azure

You cannot upload .py files directly in the Azure portal — you zip them and push via the CLI.

bash
# Navigate into the bot folder
cd ~/teams-h2o-bot/bot

# Create the zip (all 4 files must be at the root of the zip)
zip bot.zip app.py config.py h2o_bot.py requirements.txt

# Deploy to Azure App Service
az webapp deploy --name h2o-teams-bot02 --resource-group rg-h2o-teams-bot --src-path bot.zip --type zip
ℹ️
Azure unzips the files, runs pip install -r requirements.txt automatically (because SCM_DO_BUILD_DURING_DEPLOYMENT=true), then starts the server.
⚠️
Do not include the bot/ folder itself in the zip — only the files inside it. The zip bot.zip app.py ... command above does this correctly.

SymptomLikely CauseFix
Upload rejectedBad ZIP structureRun unzip -l — files must be inside a named directory
Tool not listedWrong tool type selectedRe-upload; select Local MCP Tools
Tool status ErrorMissing dependencyEnsure mcp>=1.0.0 is in requirements.txt
Server exits immediatelyImport/syntax errorRun python server.py locally and read the traceback
Tool not visible in chatEnable by Default was off⚙ Agent Configuration → Custom Tools → toggle on

SymptomLikely CauseFix

401 UnauthorizedWrong App ID or secretRe-check env vars; regenerate client secret if expired
Teams channel ErrorTerms of service not acceptedRe-enable Teams channel in Azure Bot → Channels
No tool call in responseMCP tool not enabled in h2oGPTeEnable h2o-data-utility via Agent Configuration
Upload custom app greyed outSideloading disabledTeams Admin Center → Setup Policies → Upload custom apps → On (24hr propagation)
